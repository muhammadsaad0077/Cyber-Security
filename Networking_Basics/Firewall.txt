Types:

1- Packet Filtering Firewall: First gen, basic firewall, examine basic packet info
cons: limited context, vulnerable to spoofing, no application awareness

2- Stateful inspection firewall: Second gen, maintain a state table to track active connections. Allows legitimate ( meaning that it first time check your id, next time allow you without checking id because it
remember who has visited before ) return traffic automatically.

3- Proxy Firewalls (Application-Level Gateways - Third Gen): It does a deep packet inspection, works on the application layer, examine the packet deeply that which information it contain, from where it is coming,
where it going, what it contain, is it contain something that is on the denied list, if all okay then transfer to the destination.

Cons: Performance overhead (latency), application-specific

4- Next generation firewalls: Combine traditional functions with advanced features like IPS(intrusion provision system), application control, and threat intelligence.
Cons: more complex, higher cost

Limitations
Can be bypassed by APTs/zero-days
